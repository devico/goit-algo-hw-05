## Висновки

- **Підсумок по алгоритмах.** У наших вимірах найшвидшим виявився **Боєр–Мур**, далі йде **КМП**, а **Рабін–Карп** був повільнішим через накладні витрати на хешування та перевірки збігів при колізіях.
- **За текстами окремо:**
  - **Стаття 1:** 
    - існуючий підрядок — **Боєр–Мур** (найшвидший);
    - вигаданий підрядок — **Боєр–Мур** (найшвидший).
  - **Стаття 2:** 
    - існуючий підрядок — **Боєр–Мур** (найшвидший);
    - вигаданий підрядок — **Боєр–Мур** (найшвидший).
- **Чому так:** Боєр–Мур ефективно «перестрибує» великі ділянки тексту завдяки евристиці «поганого символу», тому виграє як при наявності збігів, так і при їх відсутності. КМП має гарантовано лінійний час (побудова LPS + пошук) і часто близький до Боєра–Мура. Рабін–Карп зручний для багатьох шаблонів або попереднього хешування, але на одиничному пошуку зазвичай повільніший.
- **Загальний висновок:** Для швидкого пошуку підрядка у великому тексті практично вигідніше використовувати **Боєр–Мур**; якщо важлива гарантована лінійність і стабільність — **КМП**; **Рабін–Карп** доречний у сценаріях з багатьма шаблонами чи потребою в хешах.
